Tree 01 3.3 c-command
다음 문장의 수형도를 그리고, themselves의 antecedent가 될 수 있는 NP를 c-command 관계로 설명하시오.
The soldiers might disgrace themselves.
힌트: S → NP M VP 구조. NP = [the soldiers], M = might, VP → V NP. themselves는 VP 내 NP 자리. c-command 정의: X의 첫 번째 branching node가 Y를 dominate하면 X c-commands Y.
▸ 정답 수형도
📝 구조 분석
  • NP [the soldiers]의 첫 번째 branching node = S
  • S는 themselves를 dominate → c-command 성립 ✓
  • themselves는 anaphor → c-commanding antecedent [the soldiers] 필요
  • 복수 선행사이므로 appropriateness 조건도 충족 → 정문
Tree 02 3.3 c-command
다음 비문의 수형도를 그리고, why it is ungrammatical을 c-command로 설명하시오.
*The soldiers' behaviour might disgrace themselves.
힌트: Subject NP의 구조에 주목. [the soldiers']는 NP [the soldiers' behaviour] 내부 — NP 안에 갇혀 있어서 밖으로 c-command 불가.
▸ 정답 수형도
📝 비문 이유
  • [the soldiers']의 첫 번째 branching node = NP [the soldiers' behaviour]
  • 그 NP는 themselves를 dominate하지 않음 → c-command ✗
  • NP [the soldiers' behaviour]는 c-command 조건을 만족하지만, 단수 개체이므로 appropriateness 조건 위반
  • → 적절한 antecedent 없음 → 비문
Tree 03 NPI 'ever'
다음 문장의 수형도를 그리고, ever가 정문인 이유를 c-command로 설명하시오.
No Frenchman would ever help an Englishman.
힌트: NPI 규칙 — ever는 S의 immediate constituent(ADVP)로 붙는다. 부정어 NP [No Frenchman]이 S의 Subject로서 ever를 c-command해야 한다.
▸ 정답 수형도
📝 구조 분석
  • ever는 ADVP로 S의 immediate constituent — NPI 위치 조건 충족 ✓
  • NP [No Frenchman]의 첫 번째 branching node = S
  • S가 ADVP [ever]를 dominate → c-command 성립 ✓
  • NP [No Frenchman]이 ever보다 앞에 위치 → precede 조건 ✓
  • → 두 조건 모두 충족 → 정문
Tree 04 3.4 No Crossing Branches
다음 비문의 수형도를 그리고, No Crossing Branches Constraint 위반을 설명하시오.
*Drunks would put completely off the customers.
힌트: completely는 VP-adverb → VP의 immediate constituent여야 한다. 그러나 put off가 Phrasal Verb를 이루므로 off와 the customers가 함께 붙어야 하는데, 중간에 completely가 끼어들면 가지가 교차한다.
▸ 비문 구조 (Crossing Branches 발생)
📝 위반 설명
  • No Crossing Branches: 노드 A가 B보다 앞서면, A의 모든 자손도 B의 모든 자손보다 앞서야 한다
  • V [put]이 ADVP [completely]보다 앞서 있지만, V의 자손 P [off]는 뒤에 위치 → 위반 ✗
  • 정문은 Drunks would put the customers off completely. — off를 VP 끝으로 이동시켜 교차 해소
Tree 05 3.3 c-command 정의
TG (9), (10)의 추상 트리를 참고하여, 각 노드가 c-command하는 노드를 나열하시오.
[Abstract tree] A → B C; B → D E; C → F; F → G H
힌트: c-command 정의: X c-commands Y iff (1) X의 첫 번째 branching node가 Y를 dominate, (2) X ≠ Y, (3) X가 Y를 dominate하지 않고 Y가 X를 dominate하지 않음.
▸ 추상 트리 & c-command 관계
📝 c-command 관계 정리 (TG p.116)
  • A: c-commands nothing (루트 노드 — 위에 branching node 없음)
  • B: c-commands [C, F, G, H] — 1st branching node = A, A는 C·F·G·H dominate
  • C: c-commands [B, D, E] — 1st branching node = A
  • D: c-commands [E] — 1st branching node = B, B는 E만 dominate(D 제외)
  • E: c-commands [D] — 마찬가지
  • F: c-commands [B, D, E] — 1st branching node = A → B·D·E dominate
  • G: c-commands [H] — 1st branching node = F
  • H: c-commands [G]
