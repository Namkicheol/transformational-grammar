# 한글 용어 사용 규칙 — transformational-grammar

> **`*_study.html`, `*_concepts.html`, OX 퀴즈 해설(`why` 필드), 도식·예시 본문 등 한글이 등장하는 모든 본문에 적용되는 용어 정책.**
> 핵심: 한글 설명은 유지하되, 그 안의 학술 키워드/개념 용어는 영어 원어로 통일한다.
> 임용 관련 모든 레포(testmaster, englisheducation, phonetics-phonology, transformational-grammar, linguistics)에 공통 적용되는 정책. **원본 정책: 전역 `~/.claude/CLAUDE.md`** + **testmaster `docs/한글용어.md`**.

---

## 원칙

1. **한글 설명 자체는 유지** — 수험생을 위한 한글 풀이/요약/맥락 설명은 그대로 둔다. 한글 블록을 통째로 삭제하지 않는다.
2. **키워드 개념 용어는 영어 원어로** — 학술 용어를 한글 번역 조어로 옮기면 어색하므로 영어 원어로 표기한다.
3. **refs(합격자노트·강사 교재 MD·기출 원문) 표현 우선** — refs에 한글 표현이 있으면 그것을 쓴다. 없으면 영어 원어.
4. **혼용 금지** — 같은 개념을 한 페이지 안에서 한글/영어 혼용 금지. 한쪽으로 통일.

## 사용 OK 한글 표현 (자연스러움 / refs 또는 한국 통사론 학계 표준)

| 한글 | 영어 |
|------|------|
| 통사론 | syntax |
| 문장 / 절 / 구 | sentence / clause / phrase |
| 명사구 / 동사구 / 형용사구 | NP / VP / AP |
| 주어 / 목적어 / 보어 | subject / object / complement |
| 핵 (head) | head |
| 한정사 | determiner |
| 굴절 | inflection |
| 일치 | agreement |
| 능동·수동 | active · passive |
| 격 | case |
| 의미역 | thematic role |
| 어휘부 | lexicon |
| 어순 | word order |
| 흔적 | trace |
| 변형 | transformation |
| 도출 | derivation |
| 심층구조·표층구조 | deep structure · surface structure |
| 통제 | control |
| 결속 | binding |
| 성분 통어 | c-command |

### 자연스러운 한글 일반 설명

`이동`, `삽입`, `삭제`, `투영`, `자매 관계`, `지배 관계`, `경계 마디`, `주절·종속절` 등 — 한국 통사론 표준 한글 표기는 그대로 사용.

## 사용 금지 한글 번역 조어 → 영어 원어로 교체

> refs에 등장하지 않거나 한국어로 번역하면 어색해지는 학술 용어. 본문에서 단독 사용 시 영어 원어로 통일.

| ❌ 한글 번역 조어 | ✅ 영어 원어 |
|-----------------|------------|
| 알파 이동 | Move α (또는 Move alpha) |
| 의미역 기준 | Theta Criterion (θ-Criterion) |
| 비단계 표지 / 비표시 마디 | non-branching node 등 영어 원어 |
| 공범주 | empty category (또는 EC) |
| 외현적 이동 / 잠재적 이동 | overt movement / covert movement |
| 의문사 이동 | wh-movement |
| 핵 이동 | head movement |
| 외부 인수 / 내부 인수 | external argument / internal argument |
| 영주어 | null subject / pro |
| 인상 동사 | raising verb |
| 통제 동사 | control verb |
| 부가어 | adjunct (한글 OK이나 영어 일관 권장) |
| 구체사 | specifier |
| 수용 가능성 | acceptability (한글 OK이나 학술 본문에서는 영어 일관) |
| 가시성 조건 | Visibility Condition |
| 격 여과 | Case Filter |
| 확장 투영 원리 | Extended Projection Principle (EPP) |

> 위 표는 대표 사례. **추상적 통사 원리 이름(EPP, ECP, Projection Principle, Theta Criterion 등)은 항상 영어 원어**로 표기 (한글 글로스를 붙이려면 한 번만 정의 시 병기).

---

## 적용 패턴 예시

### Before (어색한 번역 조어 단독 사용)

```html
<p>의미역 기준에 따르면 모든 인수는 정확히 하나의 의미역을 받아야 한다.</p>
```

### After (한글 설명 유지 + 학술 키워드 영어로)

```html
<p>Theta Criterion (θ-Criterion)에 따르면 모든 argument는 정확히 하나의 thematic role을 받아야 한다.</p>
```

---

### Before (한영 병기로 중복 발생)

```js
{ ans: "O", why: "이는 의문사 이동(wh-movement)의 전형이다." }
```

### After (한글 글로스 제거)

```js
{ ans: "O", why: "이는 wh-movement의 전형이다." }
```

## 위반 즉시 수정 트리거

- `_study.html` / `_concepts.html` 본문 또는 OX 퀴즈 `why`에 위 표의 ❌ 항목이 단독 한글로 등장 → 즉시 영어 원어로 교체.
- 영어 키워드 + 한글 글로스(`c-command(성분 통어)` 등) 형태에서 한글 글로스가 어색하거나 영어와 동일 의미면 → 한글 글로스 제거.
- 표준 통사 원리 이름을 한글 번역으로 표기 → 영어 원어(EPP, ECP, Theta Criterion 등)로 즉시 교체.
- refs 어디에도 등장하지 않는 새 한글 학술 번역어를 도입 → 즉시 영어 원어로 교체.
